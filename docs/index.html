Built Different

Three principles that define Mercury.

Thinks

Mercury doesn't just execute. It orchestrates.

  • Spawns parallel sub-agents for concurrent tasks
  • Mercury Autopilot detects stuck loops by analyzing parameter diversity and success rates
  • AI self-check in Allow All mode — the model evaluates its own progress
  • 25-step agentic loop with graduated escalation

Acts

40+ built-in tools. Zero configuration.

  • Filesystem, shell, git, GitHub PRs and issues

  • Real-time progress with completion banners and token stats

Asks

Full control. Nothing happens without your say.

  • Ask Me mode: prompts for every write, command, and scope change
  • Allow All mode: auto-approve with AI self-monitoring
  • Safe command whitelist — reads never prompt
  • Directory scoping with per-session memory

Multi-Agent Orchestration

Mercury spawns parallel agents. You keep chatting.

mercury — multi-agent

Parallel Execution

Multiple tasks run simultaneously in isolated context windows.

File Locks

Reader-writer locks prevent concurrent write conflicts between agents.

Resource-Aware

Max concurrent agents auto-detected from CPU and RAM.

Non-Blocking

Keep chatting while agents work. Get notified when they finish.

Mercury Autopilot

Intelligent loop detection that knows the difference between working hard and going in circles.

Productive

High parameter diversity (> 60%) and success rate (> 70%). Mercury continues without interruption.

Suspicious

Moderate repetition detected. In Allow All: AI self-check evaluates progress. In Ask Me: prompts you.

Stuck

Low diversity and high failure rate. Mercury stops the current execution path automatically.
